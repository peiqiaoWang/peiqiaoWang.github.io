云计算基础概念

云计算的概念

云计算产生背景与概念

1、云计算驱动力

  • 技术驱动

    IT与CT融合，推动IT架构升级。

  • 需求驱动

    海量数据信息给现有设备极大压力，云计算可以把所有资源组合，按需使用，降低运营成本。

  • 商业模式转变

    企业：计算服务的运营；用户层面：为了减少成本开支

传统的计算服务却在缺陷：

一方面: IT复杂度：包括传统数据中心资源分散，无法合理利用资源，利用率低，只能达到30%，业务故障恢复时间较长，运维方面：需要多种运维工具，不同运营工具有差别，运维工程师需要手动配置机器属性，大大增加的人力和运营成本。

另一方面：现阶段计算服务，大数据处理能力差，不能有效地提升的商业嗅觉，资源需求无法弹性分配，导致资源利用率低，或无法做到动态均衡，无法支撑企业全生命周期的发展，多个数据中心分散，管理协同性差，商业注意力难以集中。

2、云计算概念

  • 商业视角： 云计算=信息电厂

    大家按需使用付费，购买信息产品到购买服务的转变

  • 技术角度

    • 狭义

      IT的基础设施的交付和使用模式，以按需易扩展，可扩展的方式，获得所需资源，包括基础架构、平台、软件。

    • 广义

      通过网络以按需易扩展的方式获得所需服务，这种服务可以使IT和软件、互联网相关的，也可以是政务云、教务云，医疗云

    云计算 = 计算/存储的网络

云计算部署模式

三种

  1. 私有云

    云计算为单一的组织拥有，并有那个组织运营。

  2. 公有云

    云服务运营商拥有云基础设施，并且为公众或者企业提供云服务。

  3. 混合云

    有两种云组成，对外还是保持一个整体

云计算商业模式

应用模式

  • IaaS(Infrastructure as a Service) 基础设施当做服务，消费者通过互联网，从基础的IT设施获得服务，例如虚拟机的租用，云盘等
  • PaaS(Platform as a Service) 平台当做服务，面向专业开发者，PaaS出现加快SaaS应用开发速度
  • SaaS(Software as a Service) 软件即服务，面向终端用户，是一种通过Internet使用软件的模式，用户无需购买，只需向软件供应商租赁使用，例如网络邮箱

行业客户对云的认识

  1. 59% 认为云计算必须演进至开放平台
  2. 75% 认为2016年IT交付将基于云进行
  3. 》50% 关心安全性
  4. 43%认为找到合适的战略伙伴以启动云战略至关重要

行业客户反应的困难与挑战

  1. 业务复杂性与平滑过渡
  2. 业务可靠性或者SLA（服务等级协议）能否支持业务，包括服务类型，服务质量
  3. 工作负载可迁移性，自动实现业务的迁移性，当服务器负载过大或出现故障时，实现热迁移
  4. 数据安全性
  5. 厂商锁定
  6. 看不到投资回报

  • 云计算服务商

    整合传统服务器，提供虚拟化服务，更经济便捷给企业、客户提供云计算服务机会；提供效率，节能减排

虚拟化技术简介

虚拟化的起源

虚拟化的概念

虚拟机的主要内容

  1. 计算虚拟化
    • CPU虚拟化
    • 内存虚拟化
    • I/O虚拟化
  2. 存储虚拟化
    • 裸设备+逻辑卷
    • 存储设备虚拟化
    • 主机存储虚拟化+文件系统
  3. 网络虚拟化
    • VPN
    • VLAN

虚拟化的本质

  1. 分区。

    单一物理服务器上运行多个虚拟机

  2. 隔离。

    每个虚拟机之间隔离，业务互不影响

  3. 封装。

    这个虚拟机执行环境封装在独立文件中，可以复制、移动来复制移动该虚拟机

  4. 独立。

    虚拟机无需任何修改，可以在任何服务器上运行，实现虚拟机的热迁移

虚拟化应用案例——热迁移

用户远程发布VM热迁移指令，同时包含虚拟网络的属性迁移，包括VLAN IP以及安全组信息，进行VM热迁移，业务毫无影响
